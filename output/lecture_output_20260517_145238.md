# What is the Transformer Model?

## Overview

The **Transformer** is a deep learning architecture introduced in the seminal 2017 paper **"Attention is All You Need"** (Vaswani et al., 2017). It fundamentally changed machine learning by replacing recurrent neural networks (RNNs) with a mechanism based entirely on **self-attention**, enabling parallel training and handling of long-range dependencies that RNNs struggled with.

> *As noted in the lectures (W5_LLM_Transformer_applications zh-Hans.pptx): "Transformer seems to provide a new type of universal model that can master knowledge more fundamental than specific task capabilities."*

---

## 1. The Problem That Transformers Solved

Before Transformers, RNNs (Recurrent Neural Networks) were the dominant architecture for sequence tasks. However, RNNs had critical issues:

| Problem | Explanation |
|---------|-------------|
| **Information loss in long sequences** | When encoding long sequences, information from early tokens gets lost or diluted |
| **Sequential training bottleneck** | RNNs process tokens one-by-one, preventing parallel computation |
| **Bias toward later inputs** | The model tends to favor more recent time steps over earlier ones |

**The Solution:** "Attention is All You Need" — a model that uses **dynamic attention weights** based on input, handles long-range dependencies, and enables fully parallel training.

*(Source: W5_LLM_Transformer_applications zh-Hans.pptx)*

## 2. Core Architecture Components

### 2.1 Self-Attention Mechanism

Each input token is transformed into three vectors via learned weight matrices:

| Vector | Symbol | Role |
|--------|--------|------|
| **Query** | Q | "What information does this token need?" |
| **Key** | K | "What information does this token contain?" |
| **Value** | V | The actual content to be passed along |

**The Process:**
1. Compute Q, K, V projections from input embeddings (via matrices WQ, WK, WV)
2. Compute attention scores: **Q × K^T** (dot product comparison)
3. Apply **softmax** to get attention weights α
4. Multiply by V to produce the output Z

**Complexity:** O(T² × d_model), where T is sequence length and d_model is the embedding dimension.

**Example from lectures (W4_LLM_Transformer.pptx):**
Input tokens: "I", "ate", "an", "apple" → x1, x2, x3, x4
- Q1 is compared with K1, K2, K3, K4 via dot product
- Softmax gives attention weights α₁,₁, α₁,₂, α₁,₃, α₁,₄
- These weights multiply V1, V2, V3, V4 to produce the output

**Coreference Resolution Example:** *"The animal didn't cross the street because it was too wide"* — Self-attention helps determine what "it" refers to (the animal or the street).

### 2.2 Multi-Head Attention

Multiple attention heads (H heads) run in parallel with separate WQ₁...WQ_H, WK₁...WK_H, WV₁...WV_H matrices. Each produces Z₁...Z_H, which are concatenated.

**Why multiple heads?** Different heads can focus on different aspects simultaneously:
- Coreference resolution
- Part of speech
- Semantic relationships
- Sentence boundaries
- Comparisons
- Context

### 2.3 Feed Forward Layer

```
Input → Norm(Z) → Feed Forward → Non-Linearity → Complex Relationships
```

- **Activation function:** GELU (Gaussian Error Linear Unit)
  - Smoother than ReLU, retains negative value information
  - Suitable for deep models like GPT
  - Improves training stability and effectiveness

### 2.4 Add & Norm (Residual Connections + Layer Normalization)

**Residual Connections:**
- Help alleviate the vanishing gradient problem in deep networks
- Gradients can flow back unchanged through residual connections
- Enable training of deeper networks

**Layer Normalization:**
- Adjusts activations to have mean 0 and variance 1
- Applied along the last dimension (feature dimension)
- Stabilizes training and has a regularization effect
- In GPT-2 and modern Transformers, applied both before and after multi-head attention

**Scale and Shift (γ and β):**
- After normalization, the model applies learnable scaling (γ) and shifting (β)
- Allows flexible adjustment of activation distribution
- Prevents all outputs from being compressed to a standard normal distribution

## 3. The Complete Architecture

The Transformer consists of **16 key components** organized into an encoder-decoder structure:

### Encoder Stack
- **Each encoder block:** Self-Attention → Add & Norm → Feed Forward → Add & Norm
- Multiple encoder blocks stacked; output of encoder i → input of encoder i+1

### Decoder Stack
- **Each decoder block:** Masked Multi-Head Attention → Add & Norm → Encoder-Decoder Attention → Add & Norm → Feed Forward → Add & Norm
- Uses **Masked Multi-Head Attention** — future tokens are masked (set to -∞ before softmax, becoming 0) so at time step t, the model can only see tokens 0 to t-1
- **Encoder-Decoder Attention:** Queries come from the decoder, Keys and Values come from the final encoder output
- Final decoder output → Linear layer → Softmax → Output Probabilities

### The 16 Components (from W4_LLM_Transformer.pptx)

| # | Component | # | Component |
|---|-----------|---|-----------|
| 1 | Tokenization | 9 | Add & Norm |
| 2 | Input Embeddings | 10 | Encoders |
| 3 | Positional Encoding | 11 | Masked Attention Mechanism |
| 4 | Query, Key, & Value | 12 | Encoder-Decoder Attention |
| 5 | Attention Mechanism | 13 | Linear |
| 6 | Self-Attention | 14 | Softmax |
| 7 | Multi-Head Attention | 15 | Decoders |
| 8 | Feed Forward | 16 | Encoder-Decoder Models |

## 4. Two Main Variants

The Transformer architecture has two primary families, as highlighted in the lectures (W5_LLM_Transformer_applications zh-Hans.pptx):

### Encoder-only (Representation) — e.g., BERT

| Property | Description |
|----------|-------------|
| **Directionality** | Bidirectional — can see all timesteps |
| **Output** | Hidden states (representations) |
| **Auto-regressive?** | No (does not inherently generate tokens) |
| **Key innovation** | MLM (Masked Language Modeling) + NSP (Next Sentence Prediction) |
| **Release** | October 2018 |
| **Scale** | BERT-Base: 110M params; BERT-Large: 340M params |

**BERT Pre-training:** Mask 60% of tokens and predict them (MLM) + predict if two sentences are consecutive (NSP). Trained on English Wikipedia (2.5B words) + BooksCorpus (800M words).

**BERT Fine-tuning:** Add task-specific module after the last encoder layer. For classification, add a feed-forward layer on the [CLS] token. For QA, train two additional vectors for answer start/end positions.

### Decoder-only (Generation) — e.g., GPT

| Property | Description |
|----------|-------------|
| **Directionality** | Auto-regressive — can only see previous timesteps |
| **Output** | Output tokens (generated text) |
| **Auto-regressive?** | Yes — previous outputs feed into next predictions |
| **Key innovation** | Causal language modeling (predict next token) |
| **Release** | June 2018 |
| **Scale** | GPT: 117M params; GPT-2: multiple sizes |

**GPT Pre-training:** Predict the next token given previous tokens (more learning signals than MLM). Trained on BooksCorpus and English Wikipedia.

**GPT Fine-tuning:** Format task-specific text as continuous flow with prompts. E.g., QA: *"Question: [X] Context: [Y] Answer:"*

*(Source: W5_LLM_Transformer_applications zh-Hans.pptx)*

## 5. The Paradigm Shift in Machine Learning

The lectures (W5_LLM_Transformer_applications zh-Hans.pptx) describe a fundamental shift in how machine learning is approached:

### Before LLMs (Traditional ML)
| Challenge | Question |
|-----------|----------|
| Feature engineering | How to design or select the best features for a task? |
| Model selection | Which model is best for which type of task? |
| Transfer learning | How to transfer knowledge from other domains with scarce labeled data? |
| Overfitting & generalization | How to balance complexity and capacity? |

### After LLMs (Transformer Era)
| New Capability | Question |
|----------------|----------|
| Pre-training & fine-tuning | How to leverage massive unlabeled data? |
| Zero-shot & few-shot learning | How to perform well on tasks the model was never trained on? |
| Prompt engineering | How to make a model understand its task through natural language? |
| Interpretability & explainability | How to understand what the model is doing? |

**Key Insight from the lectures:** The Transformer provides a **universal model** that can master knowledge more fundamental than specific task capabilities. This is the foundation of the modern LLM paradigm.

## 6. Applications Across Modalities

The lectures (W5_LLM_Transformer_applications zh-Hans.pptx) show that Transformers have expanded far beyond language:

### Language
- **BERT** (Bidirectional Encoder) — Understanding tasks
- **GPT** (Generative Decoder) — Generation tasks

### Computer Vision — Vision Transformer (ViT)
- Key breakthrough in 2020: *"An image is worth 16×16 words"*
- Divides images into fixed-size patches (16×16 pixels)
- Tokenizes each patch (linear projection of flattened patches)
- Adds position embeddings
- Feeds into a standard Transformer encoder
- Adds a learnable "class token" for classification
- **Performance:** ImageNet 88.55% Top-1, CIFAR-10 99.50%

### Audio
- Similar to vision but using spectrograms
- **Conformer:** Convolution-augmented Transformer for speech recognition
- **AST:** Audio Spectrogram Transformer

### Multi-modal
- **UniT:** Unified Transformer covering 7 tasks (object detection, visual & language reasoning, NLU)
  - Components: Image encoder + Text encoder + Joint decoder with task-level query embeddings + Specialized processing heads
- **LLaVA:** Large Language and Vision Assistant (visual instruction tuning)

### Other Modalities
- Biological/physiological signals
- And many more

## 7. Recent Developments (2023–2024) from Online Research

### Modern LLMs Built on Transformers

| Model | Year | Key Innovation |
|-------|------|----------------|
| **GPT-4** | 2023 | Multimodal (text + image input) decoder-only Transformer |
| **Llama 2/3** | 2023/2024 | Open-weight models with Grouped-Query Attention (GQA) and RoPE |
| **Mistral 7B** | 2023 | Sliding window attention, GQA |
| **Mixtral 8x7B** | 2024 | Sparse Mixture-of-Experts (MoE) — 8 experts per layer, 2 selected per token |

*Sources: https://arxiv.org/abs/2303.08774, https://arxiv.org/abs/2307.09288, https://arxiv.org/abs/2310.06825, https://arxiv.org/abs/2401.04088*

### Architectural Innovations

| Innovation | Description | Source |
|------------|-------------|--------|
| **FlashAttention** | IO-aware algorithm 2–4x faster; avoids writing full attention matrix to memory | https://arxiv.org/abs/2205.14135 |
| **FlashAttention-2** | Further optimized parallelism and warps | https://arxiv.org/abs/2307.08691 |
| **Ring Attention** | Distributes attention across devices for 100M+ token sequences | https://arxiv.org/abs/2310.01801 |
| **Grouped-Query Attention (GQA)** | Groups query heads to share key/value heads, reducing memory bandwidth | https://arxiv.org/abs/2305.13245 |
| **RoPE (Rotary Position Embedding)** | Relative position encoding via rotation matrices on Q/K vectors | https://arxiv.org/abs/2104.09864 |
| **Mamba / Mamba-2** | Selective state space models — linear-time alternative to attention | https://arxiv.org/abs/2312.00752 |
| **Diffusion Transformers (DiT)** | Replace U-Net in diffusion models with Transformers (used in Sora) | https://arxiv.org/abs/2212.09748 |
