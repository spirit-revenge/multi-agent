## 2. BERT: Bidirectional Encoder (Representation-Focused)

**Source:** `W5_LLM_Transformer_applications zh-Hans.pptx`

### Core Motivation

> "One of the biggest challenges in building language models was the lack of task-specific training data. What if we could learn an effective representation and apply it to various downstream tasks?"

This built on earlier representation learning approaches:
- **Word2vec** (2013) — learned word embeddings
- **GloVe** (2014) — learned word vectors from co-occurrence statistics

### Pre-training Corpus

| Corpus | Size |
|--------|------|
| English Wikipedia | 2.5 billion words |
| BooksCorpus | 800 million words |

### Pre-training Tasks

#### 1. MLM (Masked Language Modeling / 掩码语言建模)

BERT randomly masks tokens in a sentence and learns to predict them using bidirectional context.

```
Input:  如何 是 <MASK> 近来 今天 <SEP> <CLS>
        ↓
BERT
        ↓
Output: 如何 是 <MASK> 今天 今天 <SEP> <CLS>
        你  他们  ...  60%  20%  ...
        ↓
        Prediction Head
```

- 15% of tokens are masked
- Of those: 80% replaced with `[MASK]`, 10% replaced with random token, 10% left unchanged

#### 2. NSP (Next Sentence Prediction)

BERT learns relationships between sentence pairs.

```
... ... <SEP> ... ... <SEP> <CLS>
        ↓
BERT
        ↓
... ... <SEP> ... ... <SEP> <CLS>
is_next    not_next
  95%        5%
        ↓
        Prediction Head
```

### Model Sizes

| Variant | Parameters |
|---------|-----------|
| **BERT-Base** | 110 million |
| **BERT-Large** | 340 million |

### Fine-tuning

> "Simply add a task-specific module after the last encoder layer to map to the desired output dimensions."

## 3. GPT: Generative Pre-trained Transformer (Generation-Focused)

**Source:** `W5_LLM_Transformer_applications zh-Hans.pptx`

### Core Concept

> "A language model that can understand universal patterns?"

### Pre-training Corpus

Same as BERT: **BooksCorpus** and **English Wikipedia**

### Pre-training Task

**Next Token Prediction** (autoregressive / causal language modeling):
- Predict the next token based on all previous tokens
- The lecture notes state this provides **more learning signals than MLM**

### Model Size

| Model | Parameters |
|-------|-----------|
| **GPT-1** | 117 million |

GPT performed equally well on **GLUE** and **SQuAD** benchmarks as BERT.

### Fine-tuning

> "Format task-specific text as a continuous stream of prompts for the model to fit."

**Example — Question Answering:**
```
根据上下文回答问题。
问题：<question>
上下文：<context>
答案：<answer>
```

### Key Insights from GPT

#### 1. Effectiveness of Self-Supervised Learning
> "The model seems to learn by generating language itself, rather than relying on any specific task we design."

#### 2. Language Model as Knowledge Base
> "Generative pre-trained models perform excellently across multiple NLP tasks."

**Online References:**
- [GPT-1: Improving Language Understanding by Generative Pre-Training (2018) - OpenAI](https://cdn.openai.com/research-covers/language-unsupervised/language_understanding_paper.pdf)
- [GPT-2: Language Models are Unsupervised Multitask Learners (2019) - OpenAI](https://cdn.openai.com/better-language-models/language_models_are_unsupervised_multitask_learners.pdf)
- [GPT-3: Language Models are Few-Shot Learners (2020) - arXiv](https://arxiv.org/abs/2005.14165)

## 4. LLM Architecture (GPT-2 Decoder-Only Model)

**Source:** `W6_LLM_GPT的训练.pptx`

### Architecture Overview

> "The number of input tokens matches the number of output tokens; therefore, the first token ('Every') is not included in the output. The goal is to generate the next word 'forward'."

For the **smallest GPT-2 model**, each embedding vector has **768 dimensions**.

```
Input: "Every effort moves you"
        ↓
    Tokenization → Token Embeddings (768-dim each)
        ↓
    ┌─────────────────────────────────────┐
    │         GPT Model                   │
    │  ┌───────────────────────────────┐  │
    │  │   Masked Multi-Head Attention │  │
    │  └───────────────────────────────┘  │
    │              ↓                      │
    │         LayerNorm 2                 │
    │              ↓                      │
    │  ┌───────────────────────────────┐  │
    │  │       Feed Forward            │  │
    │  └───────────────────────────────┘  │
    │              ↓                      │
    │         LayerNorm 1                 │
    │              ↓                      │
    │          Dropout                    │
    │              ↓                      │
    │          Dropout                    │
    └─────────────────────────────────────┘
        ↓
    Final LayerNorm
        ↓
    Output: 768-dim vectors for each token
```

**Key Architectural Components:**

| Component | Description |
|-----------|-------------|
| **Token Embeddings** | Maps each token to a 768-dimensional vector |
| **Masked Multi-Head Attention** | Causal masking prevents attending to future tokens |
| **LayerNorm** | Stabilizes and accelerates training (pre-norm architecture) |
| **Feed Forward** | Two linear layers with GELU activation (e.g., 768→3072→768) |
| **Dropout** | Regularization technique to prevent overfitting |

## 5. Text Generation: Sampling Strategies

**Source:** `W6_LLM_GPT的训练.pptx`

### The Output Pipeline

```
logits (raw scores) → softmax → probability distribution → sampling → generated token
```

### Temperature Scaling

Temperature controls the "freedom" of LLM text generation:

- **Low temperature** → more deterministic, repetitive
- **High temperature** → more creative, diverse

**Mathematical formula:**
```
P(token_i) = exp(logit_i / T) / Σ_j exp(logit_j / T)
```

| Temperature | Behavior |
|-------------|----------|
| T → 0 | Greedy (always picks highest probability) |
| T < 1.0 | Sharper distribution, more focused |
| T = 1.0 | Original distribution unchanged |
| T > 1.0 | Flatter distribution, more random |
| T → ∞ | Uniform random sampling |

### Top-K Sampling

> "Instead of always choosing the token with the highest probability, we can perform probability sampling."

**Algorithm:**
1. Select only the top **K** tokens with the highest probabilities
2. Set all other token probabilities to **0**
3. Re-normalize the remaining probabilities to sum to 1
4. Sample from this truncated distribution

- **K = 1**: Greedy decoding (deterministic)
- **K = 40**: Typical value used in GPT-2
- **K = vocab size**: Full sampling from all tokens

### Top-P Sampling (Nucleus Sampling)

Mentioned in the slides but not elaborated. The core idea:

- Select the smallest set of tokens whose **cumulative probability mass exceeds threshold P**
- Dynamically adjusts the number of candidate tokens
- **P = 0.9** is a common value

- [The Curious Case of Neural Text Degeneration (2019) - arXiv](https://arxiv.org/abs/1904.09751)
- [OpenAI API Temperature Documentation](https://platform.openai.com/docs/guides/text-generation/parameter-details)
- [Hugging Face: How to Generate](https://huggingface.co/blog/how-to-generate)
