## 1. Core Definition

The **Transformer** is a deep learning architecture introduced in the landmark paper **"Attention Is All You Need"** (Vaswani et al., 2017). Its fundamental innovation is replacing recurrent (RNN) and convolutional (CNN) layers entirely with **self-attention mechanisms** to model dependencies in sequences.

- **Source (original paper):** [https://arxiv.org/abs/1706.03762](https://arxiv.org/abs/1706.03762)
- **Source (lecture):** W5_LLM_Transformer_applications zh-Hans.pptx — "Transformer, 2017年年中" (mid-2017)

The lecture notes describe this as the beginning of the LLM era, with 2018 marking the start of large language models powered by Transformers.

## 2. Architecture Overview

### 2.1 Encoder-Decoder Structure

| Component | Role | Key Sub-layers |
|-----------|------|----------------|
| **Encoder** | Representation learning (Bidirectional) | Multi-head self-attention → Feed-forward network |
| **Decoder** | Generation (Autoregressive) | Masked self-attention → Cross-attention (over encoder) → Feed-forward |

Both components use **residual connections** and **layer normalization** around each sub-layer.

- **Source (lecture):** W5_LLM_Transformer_applications zh-Hans.pptx — "BERT - 双向编码器表示" (BERT - bidirectional encoder representation) and "GPT" (generative decoder)
- **Source (web):** [https://jalammar.github.io/illustrated-transformer/](https://jalammar.github.io/illustrated-transformer/)
- **Source (web):** [https://www.tensorflow.org/text/tutorials/transformer](https://www.tensorflow.org/text/tutorials/transformer)

## 3. The Attention Mechanism Explained

### 3.1 Scaled Dot-Product Attention

The core mathematical operation is:

```
Attention(Q, K, V) = softmax(QK^T / √d_k) V
```

Where:
- **Q** = Queries
- **K** = Keys  
- **V** = Values
- **d_k** = dimension of keys (scaling factor to prevent vanishing gradients)

### 3.2 Multi-Head Attention

Multiple attention operations run in **parallel** (different "heads"), each learning different representation subspaces. Their outputs are concatenated and linearly projected.

- **Source (web):** [https://d2l.ai/chapter_attention-mechanisms-and-transformers/multihead-attention.html](https://d2l.ai/chapter_attention-mechanisms-and-transformers/multihead-attention.html)
- **Source (web):** [https://www.youtube.com/watch?v=iDulhoQ2pro](https://www.youtube.com/watch?v=iDulhoQ2pro) (3Blue1Brown visual explanation)

### 3.3 Positional Encoding

Since Transformers have **no inherent notion of sequence order**, sinusoidal positional encodings (or learned embeddings) are added to input embeddings to encode position information.

## 4. Applications Across Modalities

### 4.1 Language (NLP)

The lecture notes classify Transformer usage in language as either:
- **Representation / Encoder** → BERT (bidirectional context understanding)
- **Generation / Decoder** → GPT (autoregressive text generation)

**Key models from web research:**
- **BERT** (Google, 2018): [https://arxiv.org/abs/1810.04805](https://arxiv.org/abs/1810.04805)
- **GPT-3** (OpenAI, 2020): [https://arxiv.org/abs/2005.14165](https://arxiv.org/abs/2005.14165)
- **T5** (Google, 2019): [https://arxiv.org/abs/1910.10683](https://arxiv.org/abs/1910.10683)

### 4.2 Computer Vision — Vision Transformer (ViT)

From the lecture slides (W5_LLM_Transformer_applications zh-Hans.pptx), ViT (2020) processes images by:

1. **Tokenization:** Splitting images into fixed-size patches (e.g., 16×16 pixels)
2. **Linear projection:** Flattening each patch and projecting to embedding dimension
3. **Position embeddings:** Adding positional information
4. **Class token:** Adding a learnable "[CLS]" token for classification
5. **Standard Transformer encoder:** Processing the sequence of patch embeddings

**Performance (from lectures):**
- ImageNet Top-1 accuracy: **88.55%**
- CIFAR-10 accuracy: **99.50%**

The lectures note that ViT learns grid-like structures through position embeddings, with lower layers capturing both global and local features, and higher layers focusing on global features only.

- **Source (lecture):** W5_LLM_Transformer_applications zh-Hans.pptx — "一图胜千言：适用于大规模图像识别的Transformer模型" (A picture is worth a thousand words: Transformer for large-scale image recognition)
- **Source (web):** [https://arxiv.org/abs/2010.11929](https://arxiv.org/abs/2010.11929)
- **Source (web):** [https://arxiv.org/abs/2005.12872](https://arxiv.org/abs/2005.12872) (DETR — object detection)

### 4.3 Audio

From the lectures, audio Transformers use **spectrograms** (visual representations of audio frequencies over time) instead of raw images. Two main approaches exist:

- **Encoder-decoder variants** for sequence-to-sequence tasks
- **Pure encoder variants with CTC loss** for alignment-free training

**Key models:**
- **Conformer** — Convolution-enhanced Transformer for speech recognition
- **AST** (Audio Spectrogram Transformer)

- **Source (lecture):** W5_LLM_Transformer_applications zh-Hans.pptx — "Conformer：用于语音识别的卷积增强型Transformer" / "AST：音频频谱图Transformer"
- **Source (web):** [https://arxiv.org/abs/2005.08100](https://arxiv.org/abs/2005.08100) (Conformer)
- **Source (web):** [https://arxiv.org/abs/2104.01778](https://arxiv.org/abs/2104.01778) (AST)

### 4.4 Multimodal Transformers

#### UniT (Unified Transformer)
From the lectures, **UniT** covers **7 tasks** spanning object detection, visual-language reasoning, and natural language understanding. Its components:

| Component | Function |
|-----------|----------|
| **Image encoder** | Encodes visual input |
| **Text encoder** | Encodes language input |
| **Joint decoder** | Shared decoder with task-level query embeddings |
| **Specialized heads** | Produce final outputs per task |

- **Source (lecture):** W5_LLM_Transformer_applications zh-Hans.pptx — "多模态Transformer - UniT"

#### LLaVA (Large Language and Visual Assistant)
From the lectures, LLaVA uses **visual instruction tuning** to align vision and language modalities.

## 5. Parameter-Efficient Fine-Tuning (PEFT)

The lectures contrast two approaches:

| Approach | Full Fine-Tuning | PEFT (LoRA, BitFit) |
|----------|-----------------|---------------------|
| Resources | High compute & time | Lower compute cost |
| Training | All parameters | Selective parameters |

### LoRA (Low-Rank Adaptation)
- Freezes pre-trained weights and injects **trainable low-rank matrices** into attention layers
- Dramatically reduces trainable parameters
- **Source:** [https://arxiv.org/abs/2106.09685](https://arxiv.org/abs/2106.09685)

### BitFit
- Only fine-tunes the **bias terms** of each layer
- The lecture slide visualizes: **Blue W (weights)** = frozen (no training), **Red b (bias)** = only trained
- **Source:** [https://arxiv.org/abs/2106.10199](https://arxiv.org/abs/2106.10199)

- **Source (lecture):** W5_LLM_Transformer_applications zh-Hans.pptx — "参数高效微调技术" (Parameter-efficient fine-tuning techniques)

## 6. Current Limitations of Transformer-based LLMs

From the lecture slides (W5_LLM_Transformer_applications zh-Hans.pptx), the key problems identified are:

| Limitation | Description |
|------------|-------------|
| **True understanding vs. pattern matching** | Models may memorize rather than genuinely comprehend |
| **Factual hallucinations** | Inability to reliably follow rules, e.g., arithmetic errors |
| **Rule-following failures** | Models produce confident but incorrect outputs |

The lecture poses the question: *"注意力机制与Transformer——这是终点吗？"* (Attention mechanisms and Transformers — is this the end?) and asks: *"解决方案：？？？"* (Solutions: ???)

Additional limitations from web research:
- **Quadratic complexity:** Self-attention is O(n²) in sequence length, making long sequences expensive
- **Data hunger:** Transformers require large amounts of data to train from scratch
- **Fixed context window:** Cannot handle sequences longer than pre-trained context length

## Summary

The **Transformer** (2017) is a neural architecture based purely on **self-attention**, replacing RNNs and CNNs. It consists of **encoders** (for representation, e.g., BERT) and **decoders** (for generation, e.g., GPT). It has been successfully applied to **language, vision (ViT), audio (Conformer, AST), and multimodal tasks (UniT, LLaVA)**. Parameter-efficient fine-tuning methods like **LoRA** and **BitFit** allow adaptation with minimal compute. Despite its success, limitations include **factual hallucinations, quadratic complexity, and questions about genuine understanding** — areas of active research.
